This template is designed to be provisioned by Spotren. The app writes structured content into a stable contract instead of editing Astro components directly.

Relevant files:

- `src/content/site.json` - homepage and SEO source of truth
- `src/lib/site-schema.ts` - validates the contract
- `src/lib/site-data.ts` - loads the site content
- `src/pages/index.astro` - homepage entrypoint
- `src/content/posts/**` - optional managed posts

## Site content

`src/content/site.json` is a single JSON document. It does not use a `default` wrapper.

```json title="site.json"
{
  "head": {
    "title": "Spotren",
    "description": "Local-first business website",
    "website": "https://example.com",
    "lang": "pt-BR",
    "icon": {
      "url": "https://cdn.example.com/favicon.ico",
      "contentType": "image/x-icon"
    },
    "ogImage": {
      "url": "https://cdn.example.com/og-image.webp",
      "contentType": "image/webp"
    }
  },
  "socialLinks": [],
  "body": {
    "brand": "Spotren",
    "category": "Empresa de Software",
    "city": "São José do Rio Preto",
    "tagline": "Structured homepage copy",
    "rating": 5,
    "reviewCount": 9,
    "phone": "(17) 99999-9999",
    "address": "Street, number, city",
    "openStatus": "Aberto agora",
    "sections": []
  }
}
```

`head` drives metadata, feeds, canonical URLs, favicon references, and optional Open Graph media. `body` drives homepage content.

## Homepage sections

The homepage renders `body.sections` above the posts list. Supported section keys:

- `location`
- `reviews`
- `products`
- `services`
- `faqs`

Example:

```json title="site.json"
{
  "body": {
    "sections": [
      {
        "key": "location",
        "title": "Localização",
        "description": "Onde estamos.",
        "latitude": -20.81,
        "longitude": -49.37,
        "googleMapsUri": "https://maps.google.com/?cid=123"
      }
    ]
  }
}
```

`products` and `services` use the same catalog item fields. FAQ items need both `question` and `answer` to survive schema normalization.

## Social links

Social icons are inferred from the `name` field, so a minimal entry only needs `name` and `url`.

```json title="site.json"
{
  "socialLinks": [
    {
      "name": "instagram",
      "url": "https://instagram.com/example"
    }
  ]
}
```

## Structured data

`schemaGraph` is optional, but Spotren-managed sites may inject it so the homepage can emit JSON-LD without template edits.

## Posts

Posts live under `src/content/posts/<slug>/index.mdx`, with related media in `src/content/posts/<slug>/assets/*`.

Spotren can provision and sync these posts alongside `site.json`. The homepage still prioritizes pinned posts and falls back to the latest posts when none are pinned.
