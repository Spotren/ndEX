`ndEX` is a managed site template. In Spotren environments, the product provisions repository content and deploys the site automatically. Local development is mainly for template evolution, debugging, and contract-compatible changes.

## Prerequisites

- :link[Node.js]{id=https://nodejs.org/en/download}
- :link[pnpm]{id=https://pnpm.io/installation}
- :link[Git]{id=https://git-scm.com/}

## Local startup

```bash
pnpm install
pnpm dev
```

The local site runs at `http://localhost:4321`.

## Managed paths

When a site is provisioned through Spotren, these paths should be treated as generated or synchronized content:

- `src/content/site.json`
- `src/content/posts/**`
- `public/favicon.ico` or `public/favicon.png`

The template source can still keep example content in those paths, but managed sites may overwrite them at any time during provisioning or template sync.

## Safe template changes

Prefer changes that preserve the contract expected by `spotren-app`:

- keep `src/lib/site-schema.ts` backward compatible
- keep `src/content/site.json` as a single document with `head`, `socialLinks`, `body`, and optional `schemaGraph`
- preserve supported `body.sections[].key` values
- avoid moving managed file paths without updating the provisioner

## Generated assets

The template includes helper scripts that derive assets from structured content:

- `pnpm generate:location-maps`
- `pnpm generate:post-thumbs`
- `pnpm generate:product-thumbs`

These run automatically before `pnpm build` and `pnpm check`.

## Template updates

If you are evolving the template itself, update the template repository first and then let Spotren template sync propagate compatible changes to managed site repositories.

If you introduce a breaking content change, update both:

- the template schema and renderers in this repository
- the provisioning and sync logic in `spotren-app`
